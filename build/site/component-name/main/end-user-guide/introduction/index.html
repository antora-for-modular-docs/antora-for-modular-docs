Introduction

Starting a new Antora project

This procedure describes how to start an Antora website compatible with the Modular Documentation Initiative Guidelines, using the resources present in the https://github.com/themr0c/antora-for-modular-docs

Prerequisites

  • Docker or Podman.

Procedure

  1. Download the latest archive of the template repository https://github.com/themr0c/antora-for-modular-docs/archive/refs/heads/main.zip, extract the content to a new directory, and initialize a Git repository in that directory. Alternatively, copy the content of the archive to a directory with current content you wish to build with Antora.

  2. Organize the content. Define the project, component and guides names. Current example defines a component component-name containing one guide end-user-guide. Create the requested directory in the modules directory. See Standard File and Directory Set

  3. Edit antora.yml. See What’s antora.yml?

    ---
name: component-name (1)
title: Component Name Documentation (2)
version: master (3)
display_version: next (4)
start_page: end-user-guide:introduction.adoc (5)
nav:
  - modules/end-user-guide/nav.adoc (6)
asciidoc:
  attributes:
    # for antora
    experimental: ""
    favicon: favicon.png
    icons: font
    # for the project
    context: project-context (7)
    1 Define project name. (no spaces: results in an URL fragment)
    2 Define project title.
    3 Define version
    4 Define display_version.
    5 Define project start_page.
    6 Define navigation.
    7 Define project context. (no spaces: results in an URL fragment)

  4. Edit antora-playbook-for-development.yml. See What’s an Antora playbook?

    ---
# Use this Antora Playbook for development, to build current state in HEAD.
site:
  title: Component Name Documentation (1)
  # Disabling url on purpose to avoid htmltest crawling the live website.
  start_page: component-name:end-user-guide:introduction.adoc (2)
  robots: allow
content:
  sources:
    - url: ./
      branches: HEAD
      edit_url: "https://github.com/themr0c/antora-for-modular-docs/edit/main/{path}" (3)
output:
  destinations:
    - provider: fs
      clean: true
      path: build/site
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/master/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true
  supplemental_files: ./supplemental-ui
urls:
  html_extension_style: indexify
  redirect_facility: static
runtime:
  cache_dir: ./.cache/antora
    1 Define project title.
    2 Define project start_page.
    3 Define edit_url

  5. Edit antora-playbook-for-publication.yml

    ---
# Use this Antora Playbook for publication, to build the released version branch.
site:
  title: Example Documentation (1)
  # The url property is mandatory to generate the 404 and sitemap files.
  url: https://project.tld (2)
  start_page: end-user-guide:introduction.adoc (3)
  keys:
    google_analytics: "PLACEHOLDER" (4)
  robots: allow
content:
  sources:
    - url: ./
      branches:
        - stable (5)
        - main (5)
      edit_url: "https://github.com/themr0c/antora-for-modular-docs/edit/7.36.x/{path}" (6)
output:
  destinations:
    - provider: fs
      clean: true
      path: build/site
ui:
  bundle:
    url: https://gitlab.com/antora/antora-ui-default/-/jobs/artifacts/master/raw/build/ui-bundle.zip?job=bundle-stable
    snapshot: true
  supplemental_files: ./supplemental-ui
urls:
  html_extension_style: indexify
  redirect_facility: static
runtime:
  cache_dir: ./.cache/antora
    1 Define project title.
    2 Define peoject url
    3 Define project start_page.
    4 Define google_analytics key.
    5 Define branches to build.
    6 Define edit_url.

  6. Replace supplemental-ui/img/project-icon.svg by the project logo.

  7. Optionally, edit supplemental-ui/partials/header-content.hbs.

  8. Create an initial commit containing the result.

Verification

  1. Run preview on a local environment. The build is using same container image as the che-docs project.

    ./runnerpreview.sh
